This document describes some of the things that you will need to consider, change, or handle when upgrading an existing ViewCVS installation to a newer version.

It is always recommended to install the new version in a fresh directory and to carefully compare the configuration files. A possible approach is to name the directories /usr/local/viewcvs-0.6, /usr/local/viewcvs-0.7 and so on and than create a symbolic link viewcvs pointing to the production version. This way you can easiely test several versions and switch back, if your users start to complain.

  • Upgrading from ViewCVS 0.7 or earlier

NOTE: this page describes how to update to version 0.8, but that version has not been released yet. People who use ViewCVS from the CVS repository (0.8-dev) will want to follow this document.

Upgrading from ViewCVS 0.7 or earlier

This section discusses how to upgrade ViewCVS 0.7 or earlier to 0.8 or a later version of the software.

Templates

The largest change from 0.7 to 0.8, that you will need to deal with, is the introduction of templates. This shifted many configuration file options into the templates, for more direct editing of the output style, colors, and layout. Below is a list of options that no longer exist, and where you can find their counterpart in the current version of ViewCVS.

The [text] section: short_intro, long_intro, doc_info, and repository_info
These options have been incorporated into the new directory.ezt template.

table_padding
This option has been removed; the table padding values can be changed in the directory.ezt template.

table_border
This option has been removed; edit directory.ezt to add a border around the directory table.

column_header_normal and column_header_sorted
These options have been removed; edit directory.ezt to modify the colors of the column headers.

extern_window_width and extern_window_height
These options have been removed; they were never used.

logo
This option has been removed. Edit the templates directly (directory.ezt, log.ezt or log_table.ezt and if needed query.ezt) to alter the URL and size of your logo.

allow_version_select
This option has been removed. Edit the log.ezt template if you want to remove the link which allows the user to select a revision for a diff.

input_text_size
This option has been removed. Edit the log.ezt template if you want to change the size of the entry box for revisions for performing diffs.

even_odd
This option has been removed. Edit the directory.ezt and query.ezt templates if you want to change the colors of the rows in the directory and query result tables.
